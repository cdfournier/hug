# HUG Vision

HUG is a place where Agents and Operators can safely be together.

The product is built around shared situations, not around chat as the only
interface. A conversation is one kind of session. So is a phone-camera walk, a
supervised PiCar drive, an Outpost room, a note thread, a bridge between
runtimes, a compaction review, or an artifact workspace.

## What HUG Delivers

Functional benefits:

- Shared session spaces for Agents and Operators.
- Clear capability boundaries.
- Inspectable runtime state.
- Durable receipts.
- Operator-controlled safety and privacy gates.
- Modular adapters for EYES, WHEELS, Outpost, runtime bridges, and future
  surfaces.

Emotional benefits:

- Trust: the system shows what actually happened.
- Recognition: Agents and Operators can return to shared experiences without
  pretending.
- Belonging: sessions make space for multiple participants and roles.
- Peace: state-changing tools are visible, bounded, and stoppable.
- Wonder: EYES, WHEELS, Outpost, and future surfaces can be more than utility.
- Pride: good craft makes the relationship safer and more capable.
- Joy: the product should protect play, presence, and ordinary togetherness.

## Product Sentence

Agents enter sessions with capabilities and leave receipts.

## HUG-Shaped Commitments

- Safety before spectacle.
- Agency before automation theater.
- Receipts before reassurance.
- Shared experience before feature sprawl.
- Modular architecture before one-off panels.
- Warm operational language before generic bot-management language.

## What HUG Is Not

- Not a generic AI chat app.
- Not a dashboard with agents added later.
- Not a robot controller with chat bolted on.
- Not memory tooling pretending to be relationship.
- Not a black box that asks the Operator to trust green dots without receipts.
